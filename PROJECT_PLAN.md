# 초등학생용 MBTI 검사 웹서비스 개발 계획

## 📋 프로젝트 개요
초등학생들이 쉽고 재미있게 자신의 성격 유형을 알아볼 수 있는 MBTI 검사 웹사이트

### 타겟 사용자
- 초등학교 3~6학년 (8~12세)
- 부모님과 함께 또는 혼자서 이용 가능

### 핵심 가치
- 🎨 **쉬운 이해**: 어려운 용어 없이 쉬운 말로 설명
- 🎮 **재미있는 경험**: 게임처럼 즐겁게 참여
- 🌈 **예쁜 디자인**: 밝고 친근한 색상과 귀여운 캐릭터

## 🛠 기술 스택

### 프론트엔드
- **HTML5**: 기본 구조
- **CSS3**: 스타일링 (Flexbox, Grid 활용)
- **JavaScript (Vanilla)**: 인터랙션 및 로직
  - 외부 라이브러리 최소화로 빠른 로딩 속도 확보

### 배포
- **GitHub Pages**: 무료 호스팅
- **커스텀 도메인** (선택사항)

## 🎨 UI/UX 설계 방향

### 디자인 컨셉
1. **컬러 팔레트**
   - 파스텔 톤의 밝고 부드러운 색상
   - 각 MBTI 유형별 고유 색상 부여

2. **타이포그래피**
   - 큰 글씨 크기 (최소 16px)
   - 읽기 쉬운 한글 폰트 (예: Noto Sans KR, Black Han Sans)

3. **캐릭터**
   - 16가지 MBTI 유형별 귀여운 동물/캐릭터
   - 이모지 활용 가능

### 페이지 구성
1. **메인 페이지**
   - 간단한 소개
   - "검사 시작하기" 버튼

2. **검사 페이지**
   - 한 번에 1개 질문씩 표시
   - 진행률 바 (현재 몇 번째 질문인지)
   - 큰 버튼으로 답변 선택

3. **결과 페이지**
   - MBTI 유형 표시
   - 캐릭터 이미지
   - 쉬운 설명
   - SNS 공유 기능
   - 다시 하기 버튼

## 📊 MBTI 질문 설계 원칙

### 질문 작성 가이드
1. **쉬운 어휘**: 초등학생이 이해할 수 있는 단어 사용
2. **구체적 상황**: 추상적 개념보다 실제 일상에서 겪을 법한 상황
3. **짧은 문장**: 한 문장에 하나의 질문만
4. **양자택일**: A 또는 B 중 선택하는 형식

### 질문 예시 (각 척도별 3~4개)

#### E(외향) vs I(내향)
- 친구들과 놀이터에서 노는 게 좋아? vs 집에서 혼자 게임하는 게 좋아?
- 새 학기에 친구를 먼저 말을 걸어? vs 친구가 말을 걸 때까지 기다려?

#### S(감각) vs N(직관)
- 레고를 만들 때 설명서를 봐? vs 내가 상상한 대로 만들어?
- 이야기를 들을 때 자세한 내용이 궁금해? vs 전체적인 줄거리가 궁금해?

#### T(사고) vs F(감정)
- 친구가 틀렸을 때 바로 이야기해줘? vs 기분 나쁠까봐 조심스러워?
- 게임에서 이기는 게 중요해? vs 친구들과 재미있게 하는 게 중요해?

#### J(판단) vs P(인식)
- 숙제는 미리미리 해? vs 마감일 전날에 해?
- 계획표를 만들어서 지켜? vs 그때그때 하고 싶은 걸 해?
